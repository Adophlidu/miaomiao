# Architecture Overview

> Generated by /d:init. miaomiao — a personal bookkeeping (记账) web app.

## Tech Stack

Monorepo (pnpm workspaces + catalog, `pnpm@11.6.0`, `package.json` `type: module`).

- **Frontend** — React 19 + TanStack Router (file-based, `@tanstack/router-plugin`), Vite 8, TanStack Query, TanStack Form, Tailwind CSS v4, shadcn (`base-lyra` style) on `@base-ui/react`, `sonner` toasts (`apps/web`).
- **Backend** — Hono 4 on `@hono/node-server`, tRPC v11 via `@hono/trpc-server` (`apps/server/src/index.ts`).
- **API contract** — tRPC v11 routers (`packages/api`), shared end-to-end types with the web client.
- **Auth** — better-auth (email + password) with the Drizzle adapter (`packages/auth`).
- **Database** — PostgreSQL via Drizzle ORM (`drizzle-orm/node-postgres`) + drizzle-kit (`packages/db`).
- **Env** — `@t3-oss/env-core` + Zod, split server/web schemas (`packages/env`).
- **Validation** — Zod v4 (catalog), used for both tRPC inputs and env.

## Layering

```
apps/web (React/TanStack)  ──tRPC HTTP──▶  apps/server (Hono)
        │                                        │
        │ imports AppRouter *type* only          │ mounts
        ▼                                        ▼
   packages/api (tRPC routers)  ◀──────────  same router instance
        │                                        │
        ├── packages/auth (better-auth) ─────────┤
        └── packages/db (Drizzle + schema) ◀─────┘
   packages/env (server + web config)   packages/ui (shadcn components)
   packages/config (shared tsconfig base)
```

- The web app imports only the **type** `AppRouter` from `@miaomiao/api` (`apps/web/src/utils/trpc.ts`); no server code is bundled into the client.
- `packages/api`, `packages/auth`, and `packages/db` are server-side and consumed by `apps/server`.
- `packages/ui` is a shared shadcn component library consumed by `apps/web`.

## Data Flow (one request end-to-end)

Todo list fetch (`trpc.todo.getAll`):

1. `apps/web/src/routes/todos.tsx` — `useQuery(trpc.todo.getAll.queryOptions())`.
2. `apps/web/src/utils/trpc.ts` — `httpBatchLink` POSTs to `${VITE_SERVER_URL}/trpc` with `credentials: "include"`.
3. `apps/server/src/index.ts` — `trpcServer({ router: appRouter, createContext })` mounted at `/trpc/*`.
4. `packages/api/src/context.ts` — `createContext` resolves the better-auth session from request headers.
5. `packages/api/src/routers/todo.ts` — `todoRouter.getAll` runs `db.select().from(todo)`.
6. `packages/db/src/index.ts` — Drizzle client (`createDb()`) queries Postgres; rows return up the same path, fully typed.

Auth requests take a separate path: `app.on(["POST","GET"], "/api/auth/*", c => auth.handler(c.req.raw))` (`apps/server/src/index.ts`).

## Entry Points

- **Server** — `apps/server/src/index.ts` (builds the Hono app, mounts auth + tRPC, `serve(... port: 3000)`).
- **Web** — `apps/web/src/main.tsx` (creates the TanStack Router, mounts to `#app`); HTML shell `apps/web/index.html`.
- **DB tooling** — `packages/db/drizzle.config.ts` (drizzle-kit, schema `./src/schema`, reads `../../apps/server/.env`).

## Build / Run Commands

From the repo root (`package.json` scripts):

- `pnpm run dev` — run all workspaces (`pnpm -r dev`); web on `:3001` (`apps/web/vite.config.ts`), server on `:3000`.
- `pnpm run build` — build all workspaces (`pnpm -r build`).
- `pnpm run check-types` — typecheck all workspaces.
- `pnpm run lint` / `pnpm run format` / `pnpm run check` — Biome (authored by /d:init).
- `pnpm run db:push` / `db:generate` / `db:migrate` / `db:studio` — Drizzle schema ops (delegate to `@miaomiao/db`).

## Directory Map

| Path | Responsibility |
|---|---|
| `apps/web/` | React + TanStack Router SPA; routes, components, tRPC/auth clients. |
| `apps/server/` | Hono HTTP server; mounts better-auth handler + tRPC adapter. |
| `packages/api/` | tRPC v11 routers, context, base procedures (`publicProcedure` / `protectedProcedure`). |
| `packages/auth/` | better-auth instance (email+password) wired to the Drizzle adapter. |
| `packages/db/` | Drizzle client + Postgres schema (`auth`, `todo`) + drizzle-kit config. |
| `packages/env/` | Type-safe env via `@t3-oss/env-core` — `server.ts` and `web.ts`. |
| `packages/ui/` | Shared shadcn (`base-lyra`) component library on `@base-ui/react`. |
| `packages/config/` | Shared `tsconfig.base.json` extended by every workspace. |

## 记账 domain (implemented — spec 0001)

The bookkeeping core is the canonical full-stack exemplar (prefer it over the public `todo` example):
- **Schema:** `packages/db/src/schema/category.ts`, `transaction.ts` (per-user, FKs to `user.id`, integer-cents `amount`).
- **API:** `packages/api/src/routers/category.ts`, `transaction.ts` — all `protectedProcedure`, scoped by `ctx.session.user.id`, cross-user id → `NOT_FOUND`. Default categories seeded on first `category.list` (`packages/api/src/lib/seed-categories.ts`).
- **Web:** `apps/web/src/routes/_auth/dashboard.tsx` (summary), `categories.tsx` (CRUD), `transactions.tsx` (record + bill list); money conversion in `apps/web/src/lib/money.ts`.
- Money is stored as integer cents; the UI converts at the boundary. See `docs/conventions.md`.
