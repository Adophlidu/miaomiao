## Aesthetic & Tone

A **warm, cozy, cat-themed** (喵喵 🐱) personal bookkeeping app. Money apps usually feel cold and clinical — miaomiao deliberately feels like a friendly companion: soft, inviting, calm, and a little playful, while staying clear and trustworthy with numbers.

- **Personality:** friendly, gentle, encouraging (never stern about spending). A cat mascot/motif is a recurring, light touch — not a gimmick that gets in the way of reading balances.
- **Feel:** soft edges, warm light, comfortable breathing room. "Curling up with your finances," not "auditing a spreadsheet."
- **Guardrail:** warmth must never reduce legibility of amounts. Numbers are always high-contrast and unambiguous.

## Color System

Warm palette. Light is the hero theme; dark is a cozy "lamp-lit" companion. Both ship. Tokens map to the existing shadcn/Tailwind CSS variables in `packages/ui/src/styles/globals.css` (`--background`, `--foreground`, `--primary`, `--muted`, `--card`, `--border`, `--destructive`, etc.) — update those variables to these values rather than inventing a parallel system.

**Light theme (default warm):**
- Background: warm cream / off-white (`oklch` warm, ~`#FBF7F1`).
- Surface / card: slightly lighter warm white, soft shadow.
- Primary (brand): warm amber / honey (`~#E8A04B`) — used for primary actions and the cat motif accents.
- Secondary accent: soft terracotta / clay (`~#D98C73`).
- Foreground text: warm dark brown-charcoal (`~#3A322C`), not pure black.
- Muted text: warm taupe.
- Border: low-contrast warm beige.

**Dark theme (cozy lamp-lit):**
- Background: deep warm brown / espresso (`~#2A2320`), never cold blue-black.
- Surface / card: one step lighter warm brown.
- Primary: the same honey amber, slightly brightened for contrast.
- Foreground: warm cream.

**Semantic colors:**
- Success / income: warm-leaning green (`~#6FA86A`).
- Warning: amber (close to primary — distinguish with iconography).
- Error / destructive: warm coral-red (`~#D9645A`), not a harsh fire-engine red.
- Info: muted dusty blue (the one cool accent, used sparingly).

**Money semantics (record this convention):** income/positive = success green; expense/negative = neutral foreground (not red — spending is normal, not an error). Reserve red strictly for destructive actions and validation errors.

> Accessibility: all text/background pairs must meet **WCAG AA** (4.5:1 body, 3:1 large). Verify amber-on-cream for small text — darken the amber or use foreground text on amber fills as needed.

## Typography

- **Type family:** a friendly, rounded-but-readable sans for UI (e.g. *Nunito*, *Quicksand*, or *Inter* with slightly increased letter-spacing if a rounded face isn't added). Warmth comes from rounded terminals.
- **Numerals:** use **tabular / lining figures** for all amounts and tables so columns align (`font-variant-numeric: tabular-nums`). This is non-negotiable for a bookkeeping app.
- **Scale (comfortable):** display ~30–36px, h1 24px, h2 20px, body 16px, small 14px, caption 12px.
- **Weight:** body 400, emphasis/labels 500, headings 600–700. Amounts often 600.
- **Line-height:** 1.5 body, 1.2 headings.

## Layout & Spacing

- **Density: comfortable.** Generous but not wasteful padding.
- **Spacing scale:** Tailwind default 4px step; default component padding `p-4`/`p-6`, card gaps `gap-4`/`gap-6`.
- **Radius:** **soft and rounded** to reinforce coziness — cards `rounded-2xl` (~16px), buttons/inputs `rounded-xl` (~12px), pills `rounded-full`.
  - ⚠️ The scaffolded shadcn `base-lyra` button uses `rounded-none` (`packages/ui/src/components/button.tsx`). This **conflicts** with the cozy direction — soften component radii toward rounded as part of the first UI task.
- **Container:** content max-width ~`max-w-3xl`–`max-w-4xl`, centered, comfortable horizontal padding (`px-4`+).
- **Breakpoints:** Tailwind defaults; mobile-first (personal finance is often logged on phone).
- **Elevation:** soft, warm-tinted shadows (low blur, low opacity), not hard drop shadows. One or two elevation levels only.

## Component Rules

- **Buttons:** rounded-xl, primary = amber fill with warm-dark text, comfortable height (≥36–40px touch target). Gentle hover (slight darken + soft lift). Icon buttons keep generous hit area.
- **Inputs / forms:** rounded-xl, soft warm border, clear focus ring in primary amber. Inline validation in coral; friendly, non-blaming error copy.
- **Cards:** the primary surface for transactions, categories, summaries — `rounded-2xl`, soft shadow, warm surface color.
- **Icons:** `lucide-react` (already in stack), `1.5px`-ish stroke. A cat motif (paw, whiskers, sleeping cat) appears in empty states, the logo/header, and celebratory moments.
- **Empty states & micro-copy:** lean into the mascot — e.g. an empty ledger shows a napping cat ("还没有记录，喵～ 点上面记一笔吧"). Friendly, encouraging tone in all copy.
- **Toasts:** `sonner` (already wired, `richColors`) — success in warm green, errors in coral.

## Motion

- **Feel:** gentle and soft, never snappy/aggressive. Think "a cat stretching."
- **Duration:** 150–250ms for most transitions; 300ms for larger surface changes.
- **Easing:** ease-out / soft spring for entrances; avoid linear.
- **Delight (sparingly):** a subtle paw-tap or tail-wag on a completed action (e.g. saving a transaction). Respect `prefers-reduced-motion` — disable non-essential animation.
- The root already sets `disableTransitionOnChange` for theme switching (`apps/web/src/routes/__root.tsx`) — keep theme flips instant, animate within-page interactions.

## Theming Notes

- Both light and dark ship. `next-themes` is wired via `ThemeProvider` (`attribute="class"`, `storageKey="vite-ui-theme"`); current default is `dark`. **Recommend switching the default to `light`** (the warm cream hero theme) — confirm during the first UI task.
- All colors are CSS variables in `packages/ui/src/styles/globals.css`; theme = swap variable values under `.dark`. Do not hardcode hex in components — use the semantic Tailwind tokens (`bg-background`, `text-foreground`, `bg-primary`, etc.).

## Visual Gate Checklist (for d-ui)

A UI change is design-conformant when:
- [ ] Colors come from the warm token set above (no cold blue-black, no harsh red for normal expenses).
- [ ] Amounts use tabular numerals and meet AA contrast.
- [ ] Radii are soft/rounded (cards `2xl`, controls `xl`), consistent across the view.
- [ ] Spacing reflects comfortable density; touch targets ≥ ~40px.
- [ ] Empty/error/success states carry the friendly cat-themed voice.
- [ ] Motion is gentle and honors `prefers-reduced-motion`.
- [ ] Works in both light and dark themes.
